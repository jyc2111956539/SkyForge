# Arduino 导入 Library 的三种方法

## 一、简介

Arduino 项目经常需要使用第三方 Library，例如 OLED 显示屏库、传感器库、通信协议库等。Library 导入方式不止一种，常用方法有三类：

1. 通过 Arduino IDE 的库管理器安装。
2. 导入下载好的 `.zip` 格式库文件。
3. 将解压后的库文件夹手动复制到 `libraries` 目录。

本文整理这三种方法，并说明如何判断导入是否成功。

## 二、环境说明

本文以 Arduino IDE 2.3.8 为主，其他版本操作类似

## 三、通过库管理器安装

这种方式适合安装 Arduino 官方标准库以及库索引中已经收录的库，例如常见传感器、显示屏、网络通信等库。

### 操作步骤

1. 打开 Arduino IDE。
2. 依次点击 `项目` -> `导入库`。
3. 对于标准库直接在此选择即可，如下图
   ![](https://file.seecpp.top/seecpp/2026/05/262f307894211433a56182514877ebde.png)
   如图所示，
4. 对对于标准库没有的继续往下操作， `项目` -> `导入库` -> `管理库...`
5. 进入 `管理库` 后，在库管理器上方搜索框中输入库名或关键词，下图以搜索 `OLED` 库为例
   ![](https://file.seecpp.top/seecpp/2026/05/fcb8490170495ffa15c6f530fcf434b3.png)
6. 在搜索结果中找到目标库，确认库名称、作者和简介。
7. 选择需要的版本，点击 `安装`。
8. 如果 IDE 提示安装依赖库，建议选择一并安装。
9. 安装完成后关闭库管理器，再从 `项目` -> `导入库` 中查看库是否出现，如下图已经出现安装的OLED库
    ![](https://file.seecpp.top/seecpp/2026/05/c90fafb72956b5bab30dccdfc4f27492.png)


## 四、导入 `.zip` 格式的库

这种方式适合安装从 GitHub、厂家资料包或课程资料中下载到本地的压缩包。

### 操作步骤

1. 先准备好 Library 的 `.zip` 文件，不要提前把它解压。
2. 打开 Arduino IDE。
3. 依次点击 `项目` -> `导入库` -> `添加 .ZIP 库...`。
4. 在文件选择窗口中找到下载好的 `.zip` 文件。
5. 选中文件后点击 `打开`。
6. 等待 IDE 完成导入，底部状态栏通常会出现导入成功提示。
7. 重新打开 `项目` -> `加载库`，确认新库已经出现在列表中。


## 五、手动复制到 libraries 文件夹

这种方式适合处理已经解压好的 Library 文件夹，或者 `.zip` 导入失败的库。

### 操作步骤

1. 打开 Arduino IDE。
2. 点击 `文件` -> `首选项`。
3. 在首选项窗口中找到 `项目文件夹位置` 或 `Sketchbook location`。
4. 根据这个路径打开本地草图目录。
5. 如果目录下没有 `libraries` 文件夹，就新建一个。
6. 把解压后的库文件夹复制到 `libraries` 目录中。
7. 检查库文件夹结构，确保头文件和源码位于库的根目录或标准 `src` 目录下。
8. 关闭并重新打开 Arduino IDE。
9. 从 `项目` -> `加载库` 中确认库是否出现。


## 六、验证导入是否成功

导入成功后可以用两种方式验证。

### 6.1 查看菜单

在 `项目` -> `导入库` 中找到目标库，点击后 IDE 会自动在代码顶部插入类似下面的头文件引用：

```cpp
#include <LibraryName.h>
```

### 6.2 打开示例工程

在 `文件` -> `示例` 中查找该库自带的示例工程。如果能打开示例并正常编译，说明库基本可用。


## 七、方法选择建议

优先使用库管理器安装，因为它能自动处理版本和依赖。库管理器找不到时，再使用 `.zip` 导入。只有在压缩包导入失败、厂家只提供解压文件夹，或者需要离线拷贝时，才使用手动复制方式。
