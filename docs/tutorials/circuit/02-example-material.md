# 示例资料资料

## 资料定位

用于演示“技术资料编写规范（v1.0）”的完整写法，主题选用便携电源输入保护与基础供电链路。

## 适用范围

1. 12V/24V 直流输入场景
2. 负载电流 0.5A-3A 的中小功率系统
3. 对稳定性和可维护性有基本要求的项目

## 关键结论

1. 输入端优先做“反接保护 + 过压钳位 + EMI 抑制”三段式设计
2. 电源主拓扑先满足热与效率，再优化纹波指标
3. 文档中所有关键参数必须给单位与测试条件

## 设计/实现要点

1. 输入保护：
   TVS + 保险丝 + 反接保护 MOS
2. DC-DC 设计：
   先根据最大功耗估算电感与开关频率范围
3. 地线规划：
   功率地与信号地单点汇聚
4. 调试顺序：
   空载 -> 半载 -> 满载 -> 动态负载

## 参数与选型建议

| 项目 | 建议范围 | 说明 |
| --- | --- | --- |
| 输入 TVS 反向工作电压 | 24V 系统选 26V-33V | 需结合实际浪涌等级 |
| 开关频率 | 200kHz-600kHz | 频率越高体积小但损耗上升 |
| 输出纹波目标 | <50mVpp | 需注明测试带宽与探头接法 |
| 热设计目标 | 器件结温 <100°C | 建议保留 15%-20% 热裕量 |

## 常见问题（FAQ）

**Q1：为什么文档要先写关键结论？**  
A1：查资料场景下，用户通常先要结论，再决定是否深入细节。

**Q2：如果没有实测数据，能先发布吗？**  
A2：可以先发布，但要明确标注“仿真阶段/待实测”，避免误导。
