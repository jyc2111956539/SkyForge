# 0.96/1.3寸OLED模块设计详解（I2C/SPI四版本）

## 1. 设计目标

本项目使用统一设计思路，覆盖 4 个 OLED 模块版本：

1. 0.96寸 I2C
2. 0.96寸 SPI
3. 1.3寸 I2C
4. 1.3寸 SPI

目标是降低多型号维护成本，让不同尺寸和接口版本在供电、接口和软件适配方式上保持一致。

## 2. 版本与器件选型

### 2.1 显示控制器

- 当前硬件已测试控制器：`SSD1306`、`SH1106`
- 两种控制器在本设计中均可正常工作

### 2.2 稳压电源

- 统一使用：`SC662K-3.3V`

这样可在常见 5V 输入场景下，为 OLED 与逻辑侧提供稳定 3.3V 供电。

## 4. 硬件架构建议

### 4.1 电源部分

1. 输入端建议加保护措施（反接/限流等）。
2. 稳压输入输出侧按数据手册放置去耦电容。
3. OLED 供电与逻辑地共地。

### 4.2 I2C 版本接口

推荐：`VCC / GND / SCL / SDA`

### 4.3 SPI 版本接口

推荐：`VCC / GND / SCK / MOSI / CS / DC / RST`

## 5. 软件适配建议

1. 总线层与面板层分离：
- `bus_layer`：I2C/SPI 收发
- `panel_layer`：SSD1306/SH1106 指令与显存映射

2. 初始化按“控制器型号 + 接口”双维度区分。

## 6. PCB 设计建议

1. 稳压器与输入输出电容靠近放置。
2. 接口丝印清晰标注。
3. I2C/SPI 版本尽量保持连接器位置一致，方便结构复用。

## 7. 测试清单

1. 上电测试：3.3V 稳定性、启动显示是否正常。
2. 通信测试：I2C 100k/400k，SPI 不同主频稳定性。
3. 显示测试：全亮/全灭、文本图形混合、长时间刷新。

该方案通过统一供电（SC662K-3.3V）与双接口双尺寸覆盖，实现了较好的复用性和工程落地效率。
