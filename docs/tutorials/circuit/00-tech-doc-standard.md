# 电路设计技术资料编写规范（v1.0）

## 目的
统一资料格式，便于持续沉淀、检索与复用。

## 建议结构
1. 项目简介与应用场景
2. 关键参数与接口说明
3. 原理图与 PCB 设计说明
4. 测试结论与改进计划
5. 附件下载（原理图、PCB、BOM、数据手册）

## 命名建议
- 文件名使用英文短横线风格。
- 标题使用中文，便于站内阅读。
- 版本号使用 `v主版本.次版本`。
