# 技术资料编写规范（v1.0）

## 1. 目标

统一技术资料结构，保证内容可读、可检索、可复用，便于后续持续扩充。

## 2. 文件命名规范

1. 文件名格式：`序号-英文短名.md`，例如 `03-buck-loop-compensation.md`
2. 序号建议两位数，从 `01` 递增
3. 英文短名使用小写和连字符，不使用空格

## 3. Frontmatter 规范

必须字段：

1. `title`
2. `createTime`
3. `permalink`

建议字段：

1. `updatedAt`
2. `contentType`（固定为 `技术文档`）
3. `track`（如 `电路设计` / `嵌入式开发`）
4. `platform`（如 `电路通用` / `STM32` / `ARM`）
5. `level`（如 `专题`）
6. `tags`（3-6 个）

## 4. 正文结构规范

每篇文档建议固定以下段落顺序：

1. `资料定位`：一句话说明文档用途
2. `适用范围`：适用场景、边界条件
3. `关键结论`：先给结论，便于快速查阅
4. `设计/实现要点`：条目化说明
5. `参数与选型建议`：表格或清单
6. `常见问题（FAQ）`：至少 2 条
7. `参考链接`：数据手册、标准、工具
8. `版本记录`：更新时间和改动摘要

## 5. 内容质量要求

1. 每个结论尽量给“条件前提”，避免绝对化描述
2. 参数建议必须带单位
3. 术语首次出现时给简短解释
4. 图表没有就先留占位，不阻塞发布
5. 一篇文档聚焦一个主题，不要混多个大主题

## 6. 发布检查清单

1. 标题是否清楚表达主题
2. Frontmatter 字段是否齐全
3. 永久链接是否规范
4. 是否包含关键结论与 FAQ
5. 标签是否可用于筛选
6. 文档内链接是否可访问
